# DESIGN MANIFEST: "The Maximalist Sketchbook"
> **For Future Agents:** This document defines the visual language, component architecture, and technical constraints of the `jonny.sh` portfolio. Do not deviate from this vibe without explicit user confirmation.

## 1. Core Aesthetic: "Apple Engineering x Indie Chaos"
The design philosophy is a collision between high-precision rigid grids and chaotic, hand-drawn overlays. It represents the duality of the user (Apple Engineer + EDM/Rave Maker).

### Key Visual Pillars
1.  **The Grid (Backbone)**:
    -   Background is a subtle "Graph Paper" pattern (`background-size: 40px 40px`, `linear-gradient` grid).
    -   Everything sits on a rigid responsive grid (`.container`, `max-width: 1100px`).
    -   **Constraint**: Never break the responsive container for text, but allow artistic elements (Tape, Wires, Belts) to bleed out.

2.  **The Chaos (Overlay)**:
    -   **Rotations**: Elements are rarely straight. Use `--rotate-messy` variables (`-1.5deg`, `2deg`).
    -   **Physics**: Elements feel tacked on. Use `box-shadow` to create depth (`--shadow-paper-2`: hard offset black shadow).
    -   **Tape & Stickers**: Used to "pin" content. See `.tape` and `.sticker` classes.

3.  **Color Palette (High Voltage)**:
    -   `--bg-paper`: `#ffffff` (Clean white, not off-white).
    -   `--color-acid`: `#ccff00` (Highlighter Green/Yellow). Used for accents, marquees, and "Acid" borders.
    -   `--color-electric`: `#2b6eff` (Hyper Blue). Used for tech stack badges and active states.
    -   `--color-hot`: `#ff2a6d` (Rave Pink). Used for "Love" or "Passion" highlights.

## 2. Technical Architecture

### Component Systems
-   **FolderTabs (`FolderTabs` component)**:
    -   A skeuomorphic "File Folder" interface for Experience.
    -   **Behavior**: Tabs on top, content block below changes background color based on job (`#fafafa` vs `#e6f0ff`).
    -   **Tech Stack**: Now includes specific tech badges (`stack` array) for each role.

-   **PhotoBelt ("The Twist")**:
    -   **File**: `PhotoBeltTwist.js` / `PhotoBelt.module.css`.
    -   **Mechanism**: Two CSS belts rotating in opposite directions (`animation: scroll` vs `scrollReverse`).
    -   **Geometry**:
        -   Belt 1: `-3deg` tilt, Acid Green background.
        -   Belt 2: `+4deg` tilt, Black background, `translateZ(-50px)` for depth.
    -   **Constraint**: Images are force-constrained to `250px` width x `160px` height with `object-fit: cover`. NEVER allow raw image sizes here.

-   **Project Polaroids**:
    -   **Style**: "Photo First" approach. Images wrapped in a `.browser-frame` div (CSS dots for window controls).
    -   **Interaction**: Hover resets rotation to `0deg` and scales `1.1`.
